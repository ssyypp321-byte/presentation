ADR-0001
플랫폼 — Flutter 선택
✓ FlutterReact NativeUnity
단일 코드베이스로 iOS/Android 동시 지원. JS 브릿지 없이 60fps 보장.
ADR-0002
상태관리 — Provider 선택
✓ ProvidersetStateBLoC
게임 상태를 독립적으로 분리 가능. BLoC보다 보일러플레이트가 적어 초급 팀에 적합.
ADR-0003
데이터 저장 — Firebase 선택
✓ FirebaseHiveSQLite
로그인/회원가입 인증과 실시간 데이터 동기화가 필요했고, Flutter와 공식 연동이 잘 되어 있어서 선택했습니다.

✅ 완성된 기능
  • 자동전투 시스템 (공격속도 기반)
  • 스킬 발동 및 플레이어 개입 포인트
  • 카드 선택 → 능력치 강화
  • 장비 · 유물 시스템
  • Firebase 인증 및 데이터 저장
🔧 구현 시행착오
  • 자동전투 틱 속도 문제 → Timer 주기 최적화로 해결
  • Provider 상태 충돌 → 별도 ChangeNotifier로 분리
  • Firebase 초기화 순서 문제 → main()에서 순서 보장
🤖 AI Agent 활용
  • 기획 문서 · ADR · 아키텍처 초안 생성 후 직접 검토·수정
  • AUTHORING.md 단일 파일 부트스트랩 기법 구성
  • 화면 UI 코드 생성 후 직접 디자인 수정
📈 성능 최적화
  • 전투 루프 Timer 최소화 → 불필요한 setState 제거
  • 이미지 캐싱으로 화면 전환 지연 감소
  • Firebase 쿼리 최적화로 데이터 로딩 속도 개선
